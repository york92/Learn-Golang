# 📖 Edu Platform API 前端调试使用指南

> 各接口有严格的**角色权限**限制，请按以下流程操作，避免 `insufficient permissions` 错误。

---

## 🗺️ 角色权限速查

| 接口 | student | teacher | admin |
|------|:-------:|:-------:|:-----:|
| 注册 / 登录 | ✅ | ✅ | ✅ |
| 获取我的信息 | ✅ | ✅ | ✅ |
| 查看课程列表 / 详情 | ✅ | ✅ | ✅ |
| 创建 / 发布课程 | ❌ | ✅ | ✅ |
| 查看我的课程（师） | ❌ | ✅ | ✅ |
| 更新学习进度 | ✅ | ❌ | ✅ |
| 查询学习进度 | ✅ | ❌ | ✅ |

**核心规则：课程管理 → 用 teacher 账号；学习进度 → 用 student 账号。**

---

## 🚀 完整操作流程

### 第一阶段：注册账号

> 需要分别注册一个 **teacher** 和一个 **student**，两个角色缺一不可。

**步骤 1：注册教师账号**

1. 左侧点击 **「注册用户」**
2. 点击预设 Chip **「注册教师」**，Body 自动填入：
   ```json
   { "username": "teacher01", "email": "teacher01@test.com", "password": "123456", "role": "teacher" }
   ```
3. 点击 **Send** → 响应 `code: 0` 表示成功

**步骤 2：注册学生账号**

1. 仍在 **「注册用户」** 接口
2. 点击预设 Chip **「注册学生」**，Body 自动填入：
   ```json
   { "username": "student01", "email": "student01@test.com", "password": "123456", "role": "student" }
   ```
3. 点击 **Send** → 响应 `code: 0` 表示成功

---

### 第二阶段：教师操作（创建并发布课程）

> ⚠️ 以下步骤必须用 **teacher 账号登录**，页面顶部 Token 状态会显示当前身份。

**步骤 3：教师登录**

1. 左侧点击 **「用户登录」**
2. 点击预设 Chip **「教师登录」**：
   ```json
   { "email": "teacher01@test.com", "password": "123456" }
   ```
3. Send → 登录成功后顶部 Token 状态变为 🟢 绿色，自动保存 Token

**步骤 4：创建课程**

1. 左侧点击 **「创建课程」**
2. Body 已预填，可按需修改 `title`、`price`、`level`：
   ```json
   {
     "title": "Go 后端开发实战",
     "description": "从零到一掌握 Go 语言后端开发",
     "price": 0,
     "level": "beginner"
   }
   ```
3. Send → 响应中记录返回的课程 **`id`**（例如 `"id": 1`）

**步骤 5：发布课程**

1. 左侧点击 **「发布课程」**
2. 将路径中的 `1` 改为上一步拿到的课程 id：
   ```
   /api/v1/courses/1/publish
   ```
3. Send → 响应 `"message": "course published"` 表示成功

**步骤 6（可选）：查看我发布的课程**

1. 左侧点击 **「我的课程(师)」** → Send
2. 可看到该教师名下所有课程列表

---

### 第三阶段：学生操作（学习课程 & 更新进度）

> ⚠️ 以下步骤必须切换为 **student 账号登录**，先清除当前 Token 再重新登录。

**步骤 7：切换到学生账号**

1. 点击顶部 Token 旁的 **「清除」** 按钮，退出教师登录
2. 左侧点击 **「用户登录」**，点击预设 Chip **「学生登录」**：
   ```json
   { "email": "student01@test.com", "password": "123456" }
   ```
3. Send → 顶部 Token 自动更新为学生的 Token 🟢

**步骤 8：浏览课程**

1. 左侧点击 **「课程列表」** → Send，查看已发布的课程
2. 点击 **「课程详情」**，路径改为 `/api/v1/courses/1` → Send，查看课程章节

**步骤 9：更新学习进度**

1. 左侧点击 **「更新进度」**
2. 选择预设 Chip：
   - **「进行中」** → 记录观看了 120 秒，未完成
   - **「已完成」** → 标记该章节已完成
3. 确认 Body 中 `course_id` 和 `chapter_id` 与实际一致：
   ```json
   { "chapter_id": 1, "course_id": 1, "watched_secs": 300, "is_finished": false }
   ```
4. Send → 响应返回最新进度记录

**步骤 10：查询课程整体进度**

1. 左侧点击 **「查询进度」**
2. 路径中的 `1` 改为对应课程 id：
   ```
   /api/v1/progress/courses/1
   ```
3. Send → 响应包含：
   - `total_chapters`：课程总章节数
   - `finished_chapters`：已完成章节数
   - `percentage`：完成百分比
   - `details`：每章节详细进度

---

## ⚡ 快速重置流程

如果需要重新测试，按以下顺序重置：

```
清除 Token（顶部「清除」按钮）
  → 重新登录对应角色账号
  → 继续操作
```

如果数据库需要完全重置，停止后端服务后删除 `edu_platform.db` 文件，重启服务会自动重新建表。

## ❓ 常见报错速查

| 报错信息 | 原因 | 解决方法 |
|---------|------|---------|
| `insufficient permissions` | 角色不匹配 | 检查当前 Token 角色，参照上方权限表切换账号 |
| `missing or invalid authorization header` | 未登录或 Token 已过期 | 重新登录，Token 有效期 72 小时 |
| `invalid email or password` | 账号未注册或密码错误 | 先完成第一阶段注册 |
| `email or username already exists` | 重复注册 | 换一个 email/username，或直接登录 |
| `network request failed` | 后端未启动或跨域未配置 | 确认后端运行在 `localhost:8080`，检查 CORS 中间件 |
| `course not found` | 课程 id 不存在 | 先执行步骤4创建课程，再用返回的 id |
